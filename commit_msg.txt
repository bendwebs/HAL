feat: add SD auto-start subprocess manager and tool refresh endpoint

Stable Diffusion Auto-Start:
- New sd_process_manager.py handles starting SD as a subprocess
- Starts SD headless with --api --nowebui flags
- Auto-starts when generate_image tool is used (if SD_WEBUI_PATH configured)
- Waits for API to become ready before proceeding
- Supports both Windows (webui-user.bat) and Linux (webui.sh)

New Config Options:
- SD_WEBUI_PATH: Path to Automatic1111 webui folder
- SD_STARTUP_TIMEOUT: Seconds to wait for SD to start (default 120)

New API Endpoints:
- POST /api/images/sd/start - Manually start SD server
- POST /api/images/sd/stop - Stop SD server subprocess
- POST /api/tools/refresh - Refresh tool definitions from code

Changes:
- stable_diffusion_service.py: Added ensure_running() method
- generate_image() now auto-starts SD if path is configured
- images.py: Added start/stop endpoints and process status
- tools.py: Added refresh endpoint to sync new tools to DB
- Updated .env.example with auto-start documentation

To enable auto-start, set SD_WEBUI_PATH in your .env file.
To see the new generate_image tool, call POST /api/tools/refresh
or restart the backend.
