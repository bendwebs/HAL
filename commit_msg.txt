fix: support portable SD install with run.bat and add config

Updated sd_process_manager.py to handle various SD installation types:
- Portable installs with run.bat at root (like sd.webui packages)
- Standard installs with webui-user.bat
- Installs with webui subfolder
- Linux installs with webui.sh

Key changes:
- _find_launch_script() detects installation type
- Sets COMMANDLINE_ARGS env var for --api --nowebui flags
- Better logging during startup wait
- Increased default timeout to 180s (SD can be slow to load)

Added to .env:
- SD_WEBUI_PATH=E:\Stable_diffusion\sd.webui-1.10.1-blackwell
- SD_API_URL=http://127.0.0.1:7860
- SD_STARTUP_TIMEOUT=180
